Tester Stress & Delays

Make mt5 tester ea stress earn its name: wire MT5's native ExecutionMode delay emulation into the INI layer, run a delay ladder (ideal → fixed latencies → random), and grade the result with a deterministic robustness score an agent can gate on.

Intent

Decision this informs: the public contract for execution-realism stress testing — INI key, ladder normalization, scoring formula, verdict bands, envelope shape, and error codes — before any implementation lands.

Anti-pattern this prevents: ea.stress() today runs an ideal-execution backtest and stamps stress_delay_ms on the envelope. The one field that claims stress is the one thing the run does not apply. An honest envelope never asserts a property the run does not have.

Current vs target

Current — stress in name only

  • stress(delays_ms=50) calls single() unmodified, then tags stress_delay_ms: 50.
  • build_ea_ini() never emits ExecutionMode; every run executes instantly.
  • CLI flag --delays-ms suggests latency that is never simulated.
mt5_cli/tester/ini_builder.py:70-96 (no ExecutionMode line)
mt5_cli/tester/ea.py:285-306 (metadata-only)
mt5/cli.py:1455-1465 (--delays-ms)

Target — a graded delay ladder

  • build_ea_ini(execution_mode=...) emits a validated ExecutionMode key.
  • single(delay_ms=...) threads the value through; metadata records what was written.
  • stress(delays=[0,100,500,-1]) runs one full tester pass per rung, serially, then scores worst-case profit retention.
  • CLI: --delays 0,100,500,random; envelope: stress.v1.
Layering precedent: mt5_cli/tester/ea.py:17 (_OPT_MODES name→code map)
Stats source: mt5_cli/tester/results.py:184-256

The INI key (verified against MetaQuotes docs)

The startup-config key is ExecutionMode (metatrader5.com/en/terminal/help/start_advanced/start, checked 2026-06-10). The Strategy Tester help documents the matching UI control: network-delay emulation with No Delay, Random Delay, and Fixed Delay options (metatrader5.com/en/terminal/help/algotrading/testing, checked 2026-06-10).

INI valueMeaningCLI token
ExecutionMode=0Normal (instant) execution — the baseline rung, always run0
ExecutionMode=N (1..600000)Fixed order-execution delay of N millisecondsN
ExecutionMode=-1Random delay in order execution (per the tester help, a 0–9 second value picked per order)random
anything elseRejected: ValueError in the INI layer, INVALID_DELAYS envelope at the CLI—

Ladder normalization: dedupe · baseline 0 auto-included · order is 0, fixed delays ascending, random last · default ladder 0,100,500,random.

Robustness score — explore the contract

retention = stressed net profit / baseline net profit; score = clamp(min(retention), 0, 1) rounded to 4 dp. Worst case, not mean: the score is a gate, and a catastrophic rung must not be averaged away. Edit the numbers to see how the verdict moves.

Failure semantics: baseline fails → STRESS_BASELINE_FAILED with the failed baseline envelope under error.data.baseline (the frozen failure shape, mt5_cli/reports/envelope.py:12-16), nothing else runs. A stressed rung fails → its fail envelope ships in scenarios, scoring uses the successes, incomplete: true. Per-rung run ids fold the delay token into the id (stress-{token}-{expert}) so same-second rungs never share a run dir (mt5_cli/tester/cache.py:17-38). Schema-version style follows wake.v1 (mt5_cli/wake/wake.py:342).

Spec seed

Ownership boundaries

  • ini_builder.py owns raw MT5 codes and validation (execution_mode).
  • New mt5_cli/tester/stress.py owns token parsing, ladder normalization, scoring — pure, stdlib-only.
  • ea.py owns orchestration: serial rungs, collision-safe per-rung run ids (stress-{token}-{expert}), failure semantics, envelope assembly.
  • mt5/cli.py stays a thin flag-parsing wrapper (CONTRIBUTING architecture rule).

Public interfaces

  • build_ea_ini(..., execution_mode: int = 0)
  • single(..., delay_ms: int = 0)
  • stress(..., delays: list[int] | None = None) → stress.v1
  • mt5 tester ea stress --delays 0,100,500,random
  • Errors: INVALID_DELAYS, STRESS_BASELINE_FAILED (registered in mt5_cli/errors.py)

Build sequence

  • 1 — INI layer + tests
  • 2 — pure stress module + tests
  • 3 — orchestration; delete the stub-locking test in the same commit
  • 4 — CLI flag + error registry
  • 5 — README / AGENTS / CHANGELOG; flip spec & plan status

Non-goals

  • Stressed-optimization ranking (optimize() untouched this slice)
  • Spread/slippage/requote modeling beyond MT5's native emulation
  • Multi-symbol stress; indicator-test changes; new MCP surface
  • Any trading strategy, signal, or market opinion

Breaking change carried by the seed: delays_ms / --delays-ms are removed, not aliased — the old surface recorded stress it never applied, and an alias would preserve the lie. Full contract: docs/specs/tester-stress-delays.md; phases: docs/plans/tester-stress-delays-plan.md.
