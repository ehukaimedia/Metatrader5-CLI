Spec playground

Agent Wake Alert Bridge

MT5 alert definitions become a read-only watch-list for agents. The watcher emits structured decisions, dedupes repeats, optionally runs order dry-runs, writes audit records, and never mutates the terminal or places trades.

Intent

Read-only input

Alert definitions

mt5_cli/alert/alert.py reads MT5 alert records and does not write the binary alert store.

Decision core

Wake policy

mt5_cli/wake/wake.py validates policy, dedupes events, and emits wake.v1.

No mutation

Dry-run only

Trade templates can call the existing dry-run path. Live orders are outside mt5 alert watch.

Shipped Flow

1

Read alerts

Use mt5 alert list behavior through the read-only parser.

2

Build wake

Create wake.v1 with symbol, condition, price, and stable dedupe key.

3

Apply policy

Match by source, symbol, condition, or source text; default to notify-only.

4

Dry-run when asked

Only explicit trade templates enter order dryrun; limits run first.

5

Audit and return

Write wake_audit.v1, save state, and return one JSON envelope.

Decision Modes

notify_only

Emit and audit the wake. No trade template is required and no dry-run runs.

{
  "permission_mode": "notify_only",
  "execution": {
    "decision": "notified",
    "dryrun": null,
    "mutation": null
  }
}

Non-goals

  • No alert creation or binary alert writes.
  • No fired-alert history detection.
  • No external runtime wake hooks.
  • No mobile notification queue.
  • No live mutation from the watcher.
