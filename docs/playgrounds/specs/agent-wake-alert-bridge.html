Architectural Contracts & Non-Goals

Acceptance Criteria

  • mt5 alert watch --json must output valid JSON envelopes and always exit 0 on expected handler results to preserve the CLI contract.
  • Double-observations must be filtered out at the policy layer using a unique key schema: terminal:login:source:symbol:condition:price:timestamp.
  • No trade execution can happen on a real class account unless all three gates are active: cfg["live"] = true, environment variable MT5_LIVE=1, and the --live daemon invocation flag.
  • The ask_permission lane must verify the trade dry-run outcome, notify the agent target, and block execution until an explicit approve event is parsed.

Non-Goals

  • The CLI will never write back to the binary alerts.dat database, as the format is volatile and not round-trip validated for write commands.
  • The default MCP server remains read-only. Live mutations are blocked on the MCP interface unless a separate, explicitly configured mt5-mcp-live is armed.
  • No trading strategy or decision-making engine is built in. Wakes only carry structural parameters and template risk envelopes.
