Chart Zoom Screenshot Optimization

Add chart zoom to the CLI so agents can change visible candle density before capturing the MT5 window. Screenshot capture stays pixel-based; MT5's Save As Picture dialog is a separate exporter, not part of this slice.

Intent

Decision this informs: expose zoom as low-level chart UI control under mt5 chart zoom, then compose it with mt5 screenshot take.

Anti-pattern this prevents: tying screenshot behavior to MT5's built-in Save As Picture dialog while the current implementation captures visible pixels through mss.

mt5_cli/screenshot/screenshot.py:132-205 - mss capture path

Chart command surface

Flow

    

      
1. Locate
Find MT5 by title/class and resolve active or requested MDI child chart.

      
2. Activate
Bring that child chart to focus without changing the user's MDI layout.

      
3. Send keys
VK_ADD zooms in; VK_SUBTRACT zooms out.

      
4. Capture
Call mt5 screenshot take after the chart scale is prepared.

    

    

      The envelope reports the requested action and target chart. It does not claim
      verified scale readback because this Win32 path does not read MT5
      CHART_SCALE.
