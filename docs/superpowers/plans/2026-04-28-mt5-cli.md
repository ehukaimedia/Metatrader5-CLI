# MT5 CLI Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build `mt5`, a CLI + importable Python library that exposes MetaTrader 5 data and trading primitives as composable shell commands and structured JSON for both human operators and autonomous AI agents.

**Architecture:** Three-tier — CLI (Click groups + prompt-toolkit REPL) → Core (pure functions, JSON envelope) → Bridge (thread-safe MT5 wrapper). The bridge is the **only** module that imports `MetaTrader5`; core modules call `bridge.mt5_call(fn_name, *args)`. `core/risk.py` runs before every order regardless of caller (CLI or library import).

**Tech Stack:** Python 3.10+, MetaTrader5 ≥5.0.45, Click 8, prompt-toolkit 3, pandas 2, pandas-ta 0.3.14b, mss 9, Pillow 10, python-dateutil 2.9, pytest, unittest.mock.

**Spec:** `docs/specs/mt5-cli-spec.md` (v0.5). The plan presumes the engineer reads the spec section referenced in each task.

**Conventions used throughout:**
- Every core function returns `{"ok": True, "data": {...}}` or `{"ok": False, "error": {"code": <CODE>, "message": <str>, "mt5_retcode": <int|None>}}` (spec §6).
- All time fields are ISO-8601 UTC strings; bridge converts MT5 epoch ints at the seam.
- Core modules import only the bridge for MT5 access. The bridge re-exports MT5 constants (`ORDER_TYPE_BUY`, `TRADE_ACTION_DEAL`, `ACCOUNT_TRADE_MODE_DEMO`, etc.) so core modules never `import MetaTrader5`.
- TDD: write the failing test, run it red, implement, run it green, commit.
- Commit after each task. Conventional Commits (`feat:`, `test:`, `refactor:`, `chore:`).

## Task 2 — Bridge (mt5_backend.py)

**Spec reference:** §3 (Bridge layer contracts: symbol_select, connection lifecycle, thread safety, position close mechanism).

**Files:**
- Create: `metatrader5_cli/mt5/utils/mt5_backend.py`
- Modify: `metatrader5_cli/mt5/tests/test_core.py` (add bridge tests)

**Public API:**
- `connect(login, password, server, timeout=10000) -> None` — idempotent; sets `_initialized=True`; registers `atexit` shutdown.
- `disconnect() -> None` — direct shutdown, used in tests.
- `is_connected() -> bool`
- `mt5_call(fn_name: str, *args, **kwargs)` — `with _lock: return getattr(mt5, fn_name)(*args, **kwargs)`. The single seam through which all MT5 calls flow.
- `ensure_symbol(symbol: str) -> bool` — wraps `mt5.symbol_select(symbol, True)` under lock. Idempotent.
- `reconnect_once(cfg) -> bool` — used by error-handling code path on disconnect.
- Re-exported constants (module attributes): `TRADE_ACTION_DEAL`, `TRADE_ACTION_PENDING`, `TRADE_ACTION_SLTP`, `TRADE_ACTION_MODIFY`, `TRADE_ACTION_REMOVE`, `TRADE_ACTION_CLOSE_BY`, `ORDER_TYPE_BUY`, `ORDER_TYPE_SELL`, `ORDER_TYPE_BUY_LIMIT`, `ORDER_TYPE_SELL_LIMIT`, `ORDER_TYPE_BUY_STOP`, `ORDER_TYPE_SELL_STOP`, `ORDER_TIME_GTC`, `ORDER_TIME_SPECIFIED`, `ORDER_FILLING_FOK`, `ORDER_FILLING_IOC`, `ORDER_FILLING_RETURN`, `COPY_TICKS_ALL`, `ACCOUNT_TRADE_MODE_DEMO`, `ACCOUNT_TRADE_MODE_REAL`, `ACCOUNT_TRADE_MODE_CONTEST`, `TIMEFRAME_M1` … `TIMEFRAME_MN1` (the nine values in spec §6.3), retcode constants (10009, 10008, 10006, 10030, 10027, 10013, 10016).
- Module state: `_lock = threading.Lock()`, `_initialized = False`.

**Design notes:**
- Lock is acquired inside `mt5_call`, `ensure_symbol`, `connect`, `disconnect`, `reconnect_once`. Callers never see the lock.
- `connect` is idempotent (early-return if already initialized). REPL keeps the connection open; one-shot CLI invocations reuse the same code path.
- Auto-reconnect logic lives in `reconnect_once`; callers that hit `last_error()` indicating disconnect call it once before raising.

**Tests to add (test_core.py::TestBridge):**
- `test_connect_idempotent` — second `connect()` is a no-op.
- `test_connect_failure_raises` — when `mt5.initialize` returns False, raises `ConnectionError`.
- `test_mt5_call_dispatches_under_lock` — verify `getattr(mt5_mock, "account_info")()` was called.
- `test_ensure_symbol_calls_symbol_select_true` — verifies `(symbol, True)` args.
- `test_lock_is_threading_lock` — `isinstance(bridge._lock, type(threading.Lock()))`. (Cheap regression guard for D3.)
- `test_constants_exported` — sanity check that bridge has `ORDER_TYPE_BUY` etc.

**Commit:** `feat: thread-safe MT5 bridge with symbol_select + atexit lifecycle`

## Task 4 — Risk Envelope (core/risk.py)

**Spec reference:** §7.1 (Demo/Live gate), §7.2 (Risk envelope), §6.7 (magic resolution + strategy_id length), §8 (FIFO/hedge guard).

**Files:**
- Create: `metatrader5_cli/mt5/core/risk.py`
- Modify: `metatrader5_cli/mt5/tests/test_core.py` (TestRisk)

**Public API:**
- `resolve_magic(strategy_id: str | None, cfg: dict) -> int` — implements spec §6.7 priority: config map → auto-derive `sha256(id)[:8] % 80000 + 100000` → config default. Logs auto-derived magic on first use (use `logging` module, not print).
- `compute_volume_from_risk_pct(symbol: str, risk_pct: float, entry_price: float, sl_price: float, cfg: dict) -> float | dict` — returns lot size rounded to `volume_step`, clamped to `[volume_min, volume_max]`. `entry_price` is used (not current tick) so pending orders size correctly. Returns error envelope dict on bad inputs.
- `check_order(symbol: str, side: str, volume: float, sl: float | None, strategy_id: str | None, cfg: dict, *, is_live_intent: bool) -> dict` — the master pre-flight. Returns `{"ok": True}` or error envelope. Calls `bridge.mt5_call("account_info")`, `bridge.mt5_call("positions_get")`, `bridge.mt5_call("symbol_info_tick")`, `bridge.mt5_call("history_deals_get")` for state. **Callers must call `bridge.ensure_symbol(symbol)` before invoking `check_order`** — guard #6 (spread) reads the live tick and will return `None` if the symbol is not yet in Market Watch.
- `daily_loss(cfg: dict) -> float` — returns realized (today's deals: `profit + commission + swap`) + floating (sum of `position.profit`). UTC midnight cutoff. Negative number = loss; the cap fires when `net_daily_pnl <= -max_daily_loss` (signed comparison — avoids a profitable day falsely triggering the guard).
- `_rate_limiter` — module-level `collections.deque[float]` of recent order timestamps, sliding 60-second window (spec §7.2 D8: lives here, not in bridge).

**Guard ordering inside `check_order` (fail-fast, cheapest first):**
1. `strategy_id` length > 31 → `RISK_STRATEGY_ID_TOO_LONG`
2. Live gate: if `is_live_intent` is False but `account_info().trade_mode == ACCOUNT_TRADE_MODE_REAL` → `RISK_LIVE_GATE_BLOCKED`. `is_live_intent` is pre-computed by `_compose_live_intent` in the CLI layer: gate 1 = `cfg["live"]` (config file only, never from MT5_LIVE env), gate 2 = `--live` flag, gate 3 = `MT5_LIVE=1` env; all three must be True.
3. Symbol allowlist (if non-empty) → `RISK_SYMBOL_NOT_ALLOWED`
4. Max lot per order → `RISK_MAX_LOT_EXCEEDED`
5. SL presence + min distance → `RISK_NO_STOP_LOSS`
6. Max spread (read tick now) → `RISK_SPREAD_TOO_WIDE`
7. Hedge check (Trading.com FIFO): if existing position on symbol with opposite side → `RISK_HEDGE_BLOCKED` unless `cfg["allow_hedging"]`.
8. Max positions → `RISK_MAX_POSITIONS`
9. Min free margin % → `RISK_INSUFFICIENT_MARGIN`
10. Max daily loss (realized + floating) → `RISK_MAX_DAILY_LOSS`
11. Rate limiter (last gate, only consume a slot if all else passes) → `RISK_RATE_LIMIT`

**Tests to add (one per guard, plus integration):**
- `test_resolve_magic_uses_config_map`, `test_resolve_magic_auto_derives_in_range_100k_180k`, `test_resolve_magic_default_when_no_id`.
- `test_resolve_magic_deterministic` — same id → same magic, twice.
- `test_compute_volume_from_risk_pct_basic`, `test_compute_volume_rounds_to_volume_step`, `test_compute_volume_clamps_to_volume_min_max`.
- One `test_check_order_rejects_<GUARD>` per guard (11 tests). Each sets up the failing precondition via `mt5m` fixture, asserts the error code.
- `test_check_order_happy_path_passes`.
- `test_daily_loss_includes_realized_and_floating` — mock deals + positions, sum check.
- `test_rate_limiter_sliding_window` — fire 11 calls within 60s on `max_orders_per_minute=10`, last one rejected.
- `test_rate_limiter_does_not_consume_slot_on_earlier_failure` — verify slot is only consumed on `ok: True`.

**Commit:** `feat: core/risk envelope with 11-guard pre-order gate`

**Public API / structure:**
- `output(data: dict, as_json: bool) -> None` — the dual-mode emitter. JSON mode: `click.echo(json.dumps(data, default=str))`. Human mode: pretty-print `data["data"]` on success; `click.secho("Error [CODE]: msg", fg="red", err=True); sys.exit(<exit_code>)` on failure. Exit codes per spec §6: 0 ok, 1 user/input error (RISK_*, MT5_INVALID_*), 2 connection error.

- `_compose_live_intent(cfg, cli_live_flag) -> bool` — three-way AND of `cfg["live"]` (gate 1: config file only), `cli_live_flag` (gate 2), `os.environ.get("MT5_LIVE") == "1"` (gate 3). Gates are independent: MT5_LIVE must NOT feed into `cfg["live"]`.
- `@click.group(invoke_without_command=True)` `main(ctx, login, password, server, live, as_json, ...)` — sets `ctx.obj = {"cfg": cfg, "as_json": as_json, "live_intent": ...}`. If no subcommand, launches REPL (Task 15).
- `@main.group() config` with subcommands `show`, `save`, `test`:
  - `show` — `output({"ok": True, "data": project.mask_secrets(cfg)}, as_json)`
  - `save` — writes current effective `cfg` (after merging current CLI flags) to file
  - `test` — `bridge.connect(...)`; on success `output({"ok": True, "data": {"connected": True, "server": cfg["server"]}})`

## Task 10 — Analyze (core/analyze.py + CLI)

**Spec reference:** §6.5 (incl. v0.5 N-bar pivot algorithm with `--pivot-n 5` default).

**Files:**
- Create: `metatrader5_cli/mt5/core/analyze.py`
- Modify: `metatrader5_cli/mt5/mt5_cli.py` (add `analyze` group: `topdown`, `structure`, `bias`)
- Modify: `metatrader5_cli/mt5/tests/test_core.py`

**Public API:**
- `topdown(symbol, timeframes: list[str], bars=200) -> dict` — for each TF: fetch rates, detect swing highs/lows, classify market structure (`bullish` for HH/HL, `bearish` for LH/LL, else `neutral`). Aggregate `bias` from majority across TFs. `confluence_score` = (count of TFs agreeing with bias) / (total TFs). `notes` is a list of human-readable observations.
- `structure(symbol, timeframe, bars, pivot_n=5) -> dict` — N-bar pivot detection (spec §6.5 algorithm). Returns `swing_highs: [{time, price}]`, `swing_lows: [{time, price}]`, `support` = highest swing low below current price, `resistance` = lowest swing high above current price.
- `bias(symbol) -> dict` — calls `topdown` with default TFs (D1, H4, H1) and returns `{bias, confidence, reasoning: str}` where confidence = confluence_score and reasoning = `"\n".join(notes)`.

**CLI note — `--timeframes` parsing:** Click does not support `--option val1 val2 val3` (space-separated values in one flag). **Public API:**
- `place_market(symbol, side, *, volume=None, risk_pct=None, sl, tp=None, comment=None, strategy_id=None, magic=None, deviation=None, filling="auto", cfg, is_live_intent) -> dict`
- `place_limit(symbol, side, price, *, volume=None, risk_pct=None, sl, tp=None, expiry=None, strategy_id=None, magic=None, filling="auto", cfg, is_live_intent) -> dict`
- `place_stop(...)` — same shape as limit.
- `modify(ticket, *, sl=None, tp=None, price=None) -> dict` — auto-detects pending vs position; uses `TRADE_ACTION_SLTP` for positions, `TRADE_ACTION_MODIFY` for pending.
- `cancel(ticket) -> dict` — `TRADE_ACTION_REMOVE`.
- `poll_fill(ticket, timeout_ms=5000) -> dict` — polls `orders_get()` + `positions_get()` until found or timeout. Spec §6.7: returns `{"filled": False}` on timeout, caller decides to cancel.
- `dryrun(symbol, side, volume, sl, tp, ...) -> dict` — calls `mt5_call("order_check", request)`.
- `_resolve_filling(symbol, filling_str) -> int` — `"auto"` reads `symbol_info().filling_mode` bitmask; explicit `"FOK"/"IOC"/"RETURN"` maps directly.
- `_compute_volume(...)` — delegates to `risk.compute_volume_from_risk_pct`.

**Order pipeline (every place_* function):**
1. Validate `--volume` xor `--risk-pct` (one and only one).
2. `bridge.ensure_symbol(symbol)` — must happen first so that the spread guard inside `check_order` can read a live tick, and so `compute_volume_from_risk_pct` can call `symbol_info_tick` reliably.
3. If `risk_pct`, compute volume via `risk.compute_volume_from_risk_pct(symbol, risk_pct, entry_price, sl, cfg)` — pass the explicit `entry_price` (current tick ask/bid for market orders; limit/stop price for pending orders).
4. Call `risk.check_order(...)`. If not ok, return.
5. Resolve magic via `risk.resolve_magic`. Resolve filling.
6. Build request dict (use bridge constants).
7. `bridge.mt5_call("order_send", request)`.
8. Map retcode: 10009/10008 → `ok: True`; on 10030 include raw `filling_mode` bitmask in error message; on others → `MT5_ORDER_REJECTED` with `mt5_retcode`.
9. Return envelope with `strategy_id` echoed back.
